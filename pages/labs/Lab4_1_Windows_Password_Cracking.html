Hash File Setup & Format Identification

What You’re Doing

Downloading the Brazos Financial hash file and understanding the NTLM format — why it’s weak and why attackers love finding Windows credential stores.

Why It Matters

Understanding hash formats is fundamental to password security assessment. NTLM is a legacy Windows authentication protocol that stores passwords as unsalted MD4 hashes. “Unsalted” means identical passwords produce identical hashes — making rainbow table attacks trivially effective. This is why penetration testers celebrate when they find NTLM in an environment.

  1. Download brazos-hashes.txt from the Lab 4.1 assignment in Canvas
  2. Create a working directory:
    mkdir ~/lab4 && cd ~/lab4
  3. Move the hash file to your working directory
  4. Examine the file contents:
    cat brazos-hashes.txt
    What this does

    Concept: cat displays the full contents of a file. Here you’re examining the hash file to understand its format before attacking it.

    In your output: You should see 8 lines in username:hash format. Each hash is a 32-character hexadecimal string — this is the NTLM hash format that Windows uses to store passwords. No salt, no algorithm identifier — just the raw hash.

  5. Count the total hashes:
    wc -l brazos-hashes.txt
    What this does

    Concept: wc -l counts the number of lines in a file. In a real engagement, you’d use this to scope the attack — knowing how many accounts you’re targeting helps estimate the time and resources needed to crack them all.

John the Ripper Attack

What You’re Doing

Using John the Ripper to crack NTLM hashes with the rockyou.txt wordlist. John is a versatile, CPU-based password cracker that automatically detects hash formats.

Why It Matters

John the Ripper has been the industry standard for password auditing since 1996. It’s preinstalled on every security distribution and supports hundreds of hash formats. Understanding John’s workflow — format detection, wordlist selection, and result retrieval — is fundamental pentesting knowledge.

“Start with the wordlist. If that fails, OSINT is your friend. Don’t brute force — think like the user. People are predictable. Sports teams, kids’ names, graduation years. The password is usually simpler than you expect.”

Maya Rodriguez
Senior Penetration Tester, NEXUS Security
  1. Navigate to your working directory:
    cd ~/lab4
  2. Run John the Ripper with the rockyou wordlist against the NTLM hashes:
    john --wordlist=/usr/share/wordlists/rockyou.txt --format=NT brazos-hashes.txt
    What this does

    Concept: john is John the Ripper, one of the most widely used password cracking tools. The --wordlist flag tells it to try every password in the rockyou.txt dictionary (14 million+ real leaked passwords). The --format=NT flag specifies that these are NTLM hashes.

    In your output: Watch for lines showing cracked passwords as they’re found. John will display the username and plaintext password for each successful crack. Note the speed — NTLM hashes crack extremely fast because the algorithm was never designed to resist offline attacks.

  3. Check status if John is still running:
    john --status
    What this does

    Concept: The --status flag queries a running John session and reports progress — how many passwords have been tested, the current speed (candidates per second), and estimated time remaining. This is useful when an attack takes longer than expected.

  1. Display all cracked passwords:
    john --show --format=NT brazos-hashes.txt
    What this does

    Concept: The --show flag tells John to display all previously cracked passwords from its potfile (~/.john/john.pot). John stores results persistently so you can retrieve them later without re-running the attack.

    In your output: You should see each cracked account listed as username:password followed by a summary line showing how many were cracked out of the total. Record these — you’ll need them for your results table.

3
Hashcat Attack

What You’re Doing

Using Hashcat to crack the same NTLM hashes with GPU acceleration (or CPU fallback). Hashcat is the modern standard for high-speed password recovery.

Why It Matters

Hashcat leverages GPU parallel processing for dramatically faster cracking speeds. On modern hardware, Hashcat can test billions of NTLM hashes per second — compared to millions with CPU-only tools. Understanding both John and Hashcat gives you flexibility for different assessment scenarios.

  1. Extract raw hashes for Hashcat (it needs raw hashes, not username:hash format):
    cut -d':' -f2 brazos-hashes.txt > brazos-hashes-only.txt
    What this does

    Concept: cut splits each line on a delimiter and extracts specific fields. Here, -d':' sets the colon as the delimiter and -f2 grabs the second field (the hash). Unlike John, Hashcat expects raw hashes without the username: prefix.

    In your output: The > redirects the output to a new file. Verify with cat brazos-hashes-only.txt — you should see only 32-character hex strings, one per line, with no usernames.

Screenshot

Screenshot 6. Capture cat brazos-hashes-only.txt showing the extracted hash-only format (no usernames). This proves you understand the input format difference between John and Hashcat. Save as 06-hashes-only-format.png. VM name must be visible.

  1. Run Hashcat with a dictionary attack against the NTLM hashes:
    hashcat -m 1000 -a 0 brazos-hashes-only.txt /usr/share/wordlists/rockyou.txt -D 1
    What this does

    Concept: Hashcat is a GPU-accelerated password cracker. -m 1000 sets the hash mode to NTLM. -a 0 selects a straight dictionary attack (try each word as-is). -D 1 forces CPU mode since VMs typically lack GPU passthrough.

    In your output: Watch for the Speed.#1 line showing hashes per second (H/s). Even in CPU mode, NTLM speeds are remarkably high. Compare this number to John’s speed from Checkpoint 2 — this comparison is critical for your engagement report.

  1. Display all cracked results:
    hashcat -m 1000 brazos-hashes-only.txt --show
    What this does

    Concept: Like John, Hashcat stores cracked results in a potfile (~/.local/share/hashcat/hashcat.potfile). The --show flag displays all previously cracked hashes matched against your input file. The output format is hash:password — note this is reversed from John’s username:password format. Record these results for your comparison table.

  1. Examine Hashcat’s potfile to compare storage format with John:

warning
Common Pitfalls
  • No GPU in VM — That’s fine! Use -D 1 to force CPU mode. Speed will be lower but still works.
  • Wrong input format — Hashcat wants raw hashes only. Use cut -d':' -f2 to extract hashes from username:hash format.
  • Potfile already has results — Hashcat skips already-cracked hashes. Use --potfile-disable to ignore the potfile.
