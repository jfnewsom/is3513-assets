What You're Doing

Downloading the Brazos Financial hash file and understanding the NTLM format — why it's weak and why attackers love finding Windows credential stores.

Why It Matters

Understanding hash formats is fundamental to password security assessment. NTLM is a legacy Windows authentication protocol that stores passwords as unsalted MD4 hashes. "Unsalted" means identical passwords produce identical hashes — making rainbow table attacks trivially effective. This is why penetration testers celebrate when they find NTLM in an environment.

What You're Doing

Using John the Ripper to crack NTLM hashes with the rockyou.txt wordlist. John is a versatile, CPU-based password cracker that automatically detects hash formats.

Why It Matters

John the Ripper has been the industry standard for password auditing since 1996. It's preinstalled on every security distribution and supports hundreds of hash formats. Understanding John's workflow — format detection, wordlist selection, and result retrieval — is fundamental pentesting knowledge.

Maya Rodriguez
Senior Penetration Tester, NEXUS Security

"Start with the wordlist. If that fails, OSINT is your friend. Don't brute force — think like the user. People are predictable. Sports teams, kids' names, graduation years. The password is usually simpler than you expect."

Step-by-Step Instructions

Navigate to your working directory:

cd ~/lab4

Run John the Ripper with the rockyou wordlist:

Note the start time when the attack begins — you will need the total cracking time for your documentation.

john --wordlist=/usr/share/wordlists/rockyou.txt --format=NT brazos-hashes.txt
▶ What this does

Flag breakdown:

  • --wordlist=... — Dictionary attack mode — tries every password in rockyou.txt
  • --format=NT — Tells John these are NTLM hashes (Windows NT LAN Manager)
  • brazos-hashes.txt — The target file containing username:hash pairs

In your output: John will display cracked passwords as it finds them. Not all 8 will crack — that is by design. Press any key to see progress.

What You're Doing

Using Hashcat to crack the same NTLM hashes with GPU acceleration (or CPU fallback). Hashcat is the modern standard for high-speed password recovery.

Why It Matters

Hashcat leverages GPU parallel processing for dramatically faster cracking speeds. On modern hardware, Hashcat can test billions of NTLM hashes per second — compared to millions with CPU-only tools. Understanding both John and Hashcat gives you flexibility for different assessment scenarios.

Step-by-Step Instructions

Extract raw hashes for Hashcat:

Hashcat expects raw hashes without usernames. Use cut to extract just the hash column:

cut -d':' -f2 brazos-hashes.txt > brazos-hashes-only.txt
▶ What this does

Flag breakdown:

  • -d':' — Delimiter — split each line at the colon character
  • -f2 — Field 2 — grab the second column (the hash)
  • > brazos-hashes-only.txt — Redirect output to a new file

In your output: Verify with cat brazos-hashes-only.txt — you should see 8 lines of 32-character hex strings, no usernames.

Run Hashcat dictionary attack:

Note the start time and watch for the cracking speed (H/s = hashes per second).

hashcat -m 1000 -a 0 brazos-hashes-only.txt /usr/share/wordlists/rockyou.txt
▶ What this does

Flag breakdown:

  • -m 1000 — Hash mode 1000 = NTLM
  • -a 0 — Attack mode 0 = dictionary (straight wordlist)
  • brazos-hashes-only.txt — Your extracted hash file
  • /usr/share/wordlists/rockyou.txt — The wordlist to try

In your output: Watch for the Speed line — compare this to John's speed from Checkpoint 2.

Pro Tip: No GPU in your VM? That is expected. Add -D 1 to force CPU mode: hashcat -m 1000 -a 0 brazos-hashes-only.txt /usr/share/wordlists/rockyou.txt -D 1

Common Pitfalls
  • No GPU in VM: That's fine! Use -D 1 to force CPU mode. Speed will be lower but still works.
  • Wrong input format: Hashcat wants raw hashes only. Use cut -d':' -f2 to extract hashes from username:hash format.
  • Potfile already has results: Hashcat skips already-cracked hashes. Use --potfile-disable to ignore the potfile.

What Red Team Learns
  • Credential Access: NTLM hashes are a penetration tester's goldmine — fast to crack and often lead to lateral movement
  • OSINT Enhancement: Dictionary attacks are step one. OSINT-informed wordlists dramatically improve success rates against targeted accounts
  • Tool Efficiency: Knowing when to use John vs. Hashcat depends on the scenario — both belong in your toolkit
What Blue Team Learns
  • Defense in Depth: NTLM is architecturally weak. Defenders must compensate with MFA, monitoring, and credential hygiene programs
  • Password Policy Design: Complexity requirements alone don't stop OSINT-informed attacks. Passphrases and blocklists matter more
  • Credential Monitoring: If attackers can extract hashes, you've already lost. Focus on preventing initial access and detecting extraction
The Purple Team Connection

Password cracking bridges offense and defense perfectly. Attackers use these tools to exploit weak credentials — the same tools defenders use to audit password strength proactively. Understanding cracking speeds informs both attack timelines and defensive response windows. A password that cracks in seconds requires different defensive controls than one that takes hours. This is why both red and blue teams need hands-on cracking experience.
