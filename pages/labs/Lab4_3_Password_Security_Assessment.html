    • Each recommendation specific: what to do, what effort required, what priority window
    • Differentiated by client (some recs apply only to Brazos’s Windows environment, others to Alamo’s Linux environment)
    • Aligned with findings (each recommendation maps to one or more findings)
    • Where applicable, indicate which control framework the recommendation aligns with (NIST SP 800-63B, CIS Controls v8, NIST CSF)

Platform Comparison Analysis

What You’re Doing

The core analytical work of your report — comparing Windows NTLM and Linux SHA-512 password security architectures using your own cracking data.

Why It Matters

This is THE teaching moment of Module 4. You have empirical evidence showing that the same password cracks thousands of times faster on Windows than Linux. The difference isn’t password complexity — it’s defensive architecture. This is why security professionals argue about platform choices.

  1. Create a speed comparison table using your actual data:
  2. Answer these four analysis questions in your report:
    • Why did NTLM hashes crack faster than SHA-512?
    • What defensive mechanisms does Linux use that Windows NTLM lacks?
    • Which client faces greater credential risk? Why?
    • What would you recommend each client prioritize?
  3. Explain the technical reasons for speed differences:
    • NTLM: Single-pass MD4, no salt, no iterations
    • SHA-512: Salted, 5000+ rounds of key derivation
