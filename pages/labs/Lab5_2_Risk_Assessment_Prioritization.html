What You're Doing

Scoring each vulnerability from Lab 5.1 using the CVSS 3.1 calculator.

Why It Matters

CVSS provides a standardized way to communicate vulnerability severity. It's the common language between security teams, vendors, and clients.

Step-by-Step Instructions

  1. Access the CVSS Calculator: NVD calculator at nvd.nist.gov or FIRST calculator at first.org
  2. For Each Vulnerability: Look up the CVE (if available) or calculate based on characteristics
  3. Document Components: Attack Vector, Attack Complexity, Privileges Required, User Interaction, Scope, C/I/A Impact
  4. Create a Comprehensive Table: CVE ID (if applicable), Vulnerability Name, Base Score, Severity Rating

What You're Doing

Building a 5×5 risk matrix plotting vulnerabilities by likelihood and impact.

Why It Matters

Executives understand "red quadrant" better than "CVSS 9.8." A risk matrix translates technical assessments into business language.

Step-by-Step Instructions

  1. Define Likelihood Levels (1-5): Rare → Unlikely → Possible → Likely → Almost Certain
  2. Define Impact Levels (1-5): Negligible → Minor → Moderate → Major → Catastrophic
  3. Calculate Risk Scores: Risk = Likelihood × Impact. Critical (15-25), High (10-14), Medium (5-9), Low (1-4)
  4. Create Visual Matrix: 5×5 grid, color-coded by risk level

Common Pitfalls
  • Inconsistent scales: Use the same 1-5 scale for both likelihood and impact
  • Missing color coding: Visual distinction makes the matrix actionable
  • No thresholds defined: Document what score ranges mean (critical, high, medium, low)
