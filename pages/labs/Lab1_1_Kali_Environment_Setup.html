Download and Install VirtualBox

What You’re Doing

Installing VirtualBox, a free hypervisor that allows you to run virtual machines on your computer.

Why It Matters

Virtual machines allow you to run a complete operating system inside your existing OS. This is how security professionals maintain isolated environments for testing tools, malware analysis, and client engagements without risking their primary system.

  1. Navigate to virtualbox.org/wiki/Downloads
  2. Download the installer for your host operating system (Windows, macOS, or Linux)
  3. Run the installer with default settings
  4. Restart your computer if prompted
  5. Launch VirtualBox to verify installation
warning
Common Pitfalls
  • macOS users: You may need to allow the kernel extension in System Preferences > Security & Privacy
  • Windows users: Enable Hyper-V/virtualization in BIOS if VirtualBox won't start VMs

Download Kali Linux ISO

What You’re Doing

Downloading the Kali Linux installer image (~4GB) from the official source.

Why It Matters

Kali Linux is a Debian-based distribution designed specifically for penetration testing and security auditing. It comes pre-loaded with hundreds of security tools that you'll use throughout the semester for both offensive and defensive work.

  1. Navigate to kali.org/get-kali
  2. Select Installer Images
  3. Choose the correct architecture: 64-bit (amd64): For Intel/AMD processors Apple Silicon (arm64): For M1/M2/M3 Macs using UTM
  4. Download the ISO file (approximately 4GB)
  5. Note the download location — you'll need this for the next checkpoint

Create and Configure VM

What You’re Doing

Creating a new virtual machine in VirtualBox with settings optimized for Kali Linux.

Why It Matters

Proper VM configuration affects performance, stability, and functionality. The VM name is especially important — it appears in your terminal prompt and must be visible in screenshots for assignment verification.

  1. In VirtualBox Manager, click New
  2. Configure the VM name using this exact format: kali-lastname-studentID Example: kali-newsom-abc123
  3. Set Type: Linux, Version: Debian (64-bit)
  4. Allocate RAM: 4096 MB minimum (8192 MB recommended if available)
  5. Create a virtual hard disk: VDI format, Dynamically allocated, 80 GB
  6. Before starting, go to Settings > System > Processor: Allocate 2+ CPUs
  7. Go to Settings > Storage: Click the empty disk under Controller: IDE, then click the disk icon and choose your Kali ISO
error
Critical: VM Naming Convention

Your VM name must follow the format kali-lastname-studentID. This name appears in your terminal prompt and verifies that screenshots are your own work. Incorrect naming will result in unverifiable screenshots.

Install Kali Linux

What You’re Doing

Running the Kali Linux installer inside your virtual machine to create a functional security testing environment.

Why It Matters

Kali Linux comes pre-loaded with hundreds of security tools. A proper installation ensures all tools work correctly and the system is stable for the entire semester.

  1. Start the VM in VirtualBox Manager
  2. Select Graphical Install from the boot menu
  3. Choose your language, location, and keyboard layout
  4. Set hostname to match your VM name: kali-lastname-studentID
  5. Leave domain name blank
  6. Create your user account (remember these credentials!)
  7. Use Guided - use entire disk for partitioning
  8. Select All files in one partition
  9. Confirm and write changes to disk
  10. Wait for installation to complete (15-25 minutes)
  11. Reboot when prompted

Install and Verify Docker

What You’re Doing

Installing Docker, the containerization platform we'll use to deploy vulnerable lab targets throughout the course.

Why It Matters

Docker lets us spin up isolated, vulnerable systems safely. Each module's client engagement uses Docker containers — without it, you can't complete the labs.

  1. Update your package lists: sudo apt update ▶ What this does Concept: apt update refreshes the list of available packages from the repositories. It does not install anything — it just checks what’s available. Always run this before installing new software so you get the latest versions.
    sudo apt update
    What this does

    Concept: apt update refreshes the list of available packages from the repositories. It does not install anything — it just checks what's available. Always run this before installing new software so you get the latest versions.

  2. Install Docker and Docker Compose: sudo apt install -y docker.io docker-compose ▶ What this does Concept: apt install downloads and installs packages. The -y flag automatically answers “yes” to the confirmation prompt. docker.io is the container engine; docker-compose lets you define multi-container environments in a single file. We’ll use both throughout the course.
    sudo apt install -y docker.io docker-compose
    What this does

    Concept: apt install downloads and installs packages. The -y flag automatically answers "yes" to the confirmation prompt. docker.io is the container engine; docker-compose lets you define multi-container environments in a single file.

  3. Enable Docker to start automatically and start it now: sudo systemctl enable docker --now ▶ What this does Concept: systemctl enable tells the system to start Docker automatically on every boot. The --now flag also starts it immediately so you don’t have to reboot first.
    sudo systemctl enable docker --now
    What this does

    Concept: systemctl enable tells the system to start Docker automatically on every boot. The --now flag also starts it immediately so you don't have to reboot first.

  4. Add your user to the Docker group: sudo usermod -aG docker $USER ▶ What this does Concept: usermod -aG appends (-a) your user to a group (-G). By default, Docker commands require root access. Adding yourself to the docker group lets you run Docker without sudo every time.
    sudo usermod -aG docker $USER
    What this does

    Concept: usermod -aG appends your user to a group. By default, Docker commands require root access. Adding yourself to the docker group lets you run Docker without sudo every time.

lightbulb
Pro Tip

After adding yourself to the docker group, you must log out and back in (or reboot) for the group membership to take effect. Until then, you'll need sudo for docker commands.

  1. Check the Docker version: docker --version ▶ What this does In your output: You should see the installed Docker version number. If you get “command not found” or a permission error, the installation didn’t complete correctly or you haven’t logged out and back in yet.
    docker --version
    What this does

    In your output: You should see the installed Docker version number. If you get "command not found" or a permission error, the installation didn't complete correctly or you haven't logged out and back in yet.

  2. Run the Docker test container: docker run hello-world ▶ What this does Concept: docker run pulls an image from Docker Hub and runs it as a container. The hello-world image is a tiny test container that prints a success message. If you see “Hello from Docker!” — Docker is fully working.
    docker run hello-world
    What this does

    Concept: docker run pulls an image from Docker Hub and runs it as a container. The hello-world image is a tiny test container that prints a success message. If you see "Hello from Docker!" — Docker is fully working.

What You’re Doing

Running a series of commands to verify your environment is properly configured and all required tools are available.

Why It Matters

    hostname
    What this does

    In your output: This should show kali-lastname-studentID. If it shows something else, you need to fix it before proceeding.

  2. Display kernel and OS information: uname -a ▶ What this does Concept: uname -a prints all system information: kernel name, hostname, kernel version, architecture. The -a flag means “all.” This is a standard first step when documenting an engagement environment.
    uname -a
    What this does

    Concept: uname -a prints all system information: kernel name, hostname, kernel version, architecture. The -a flag means "all." This is a standard first step when documenting an engagement environment.

info

Part B: Network Connectivity

  1. Display your network interfaces and IP addresses: ip addr ▶ What this does Concept: ip addr lists all network interfaces and their assigned IP addresses. Look for your eth0 interface — it should have an IP address assigned by VirtualBox’s NAT adapter (typically 10.0.2.x). No IP means no network connectivity.
    ip addr
    What this does

    Concept: ip addr lists all network interfaces and their assigned IP addresses. Look for your eth0 interface — it should have an IP address assigned by VirtualBox's NAT adapter (typically 10.0.2.x).

  2. Test internet connectivity: ping -c 3 8.8.8.8 ▶ What this does Concept: ping sends ICMP echo requests to test network reachability. The -c 3 flag limits it to 3 packets (otherwise it runs forever). 8.8.8.8 is Google’s public DNS. If all 3 packets come back, you have internet connectivity.
    ping -c 3 8.8.8.8
    What this does

    Concept: ping sends ICMP echo requests to test network reachability. The -c 3 flag limits it to 3 packets. 8.8.8.8 is Google's public DNS. If all 3 packets come back, you have internet connectivity.
