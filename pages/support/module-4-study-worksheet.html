Attack Fundamentals

Define each term. If you can’t explain it simply, you don’t understand it well enough.

Attack Categories

  • Target of opportunity — any vulnerable system found
  • Targeted attack — specific organization chosen deliberately
  • Active attack — modifies data or systems
  • Passive attack — observes without modification

Spoofing & Impersonation

  • Spoofing — making traffic appear from different source
  • IP spoofing — falsifying source IP address
  • MAC spoofing — falsifying MAC address
  • Email spoofing — forging sender address
  • Caller ID spoofing — falsifying phone number
2
Malware

Malware Types

  • Virus — requires host file, user action to spread
  • Worm — self-replicating, no user action needed
  • Trojan — disguised as legitimate software
  • RAT (Remote Access Trojan) — provides backdoor access
  • Rootkit — hides at OS/kernel level, very difficult to detect
  • Bootkit — rootkit that infects boot process
  • Ransomware — encrypts files, demands payment
  • Spyware — monitors user activity
  • Keylogger — captures keystrokes
  • Adware — displays unwanted advertisements
  • Botnet — network of compromised systems (zombies)
  • Logic bomb — triggers on specific condition or date
  • Fileless malware — lives in memory only
  • PUP (Potentially Unwanted Program) — gray area software

Malware Techniques

  • Polymorphic — changes code to avoid detection
  • Metamorphic — completely rewrites itself
  • Armored — makes analysis difficult
  • Packing — compresses/encrypts to hide signatures
  • Shimming — inserting code between driver and OS
3
Social Engineering

Phishing Variants

  • Phishing — mass email attack, generic targets
  • Spear phishing — targeted at specific individual
  • Whaling — targeting executives
  • BEC (Business Email Compromise) — impersonating CEO for wire transfer
  • Vishing — voice phishing via phone
  • Smishing — SMS phishing
  • Pharming — redirecting to fake website via DNS

Social Engineering Techniques

  • Pretexting — fabricated scenario to extract information
  • Baiting — leaving infected media (USB drives)
  • Quid pro quo — offering something in exchange for info
  • Tailgating/Piggybacking — following through secure doors
  • Shoulder surfing — watching someone enter credentials
  • Dumpster diving — searching trash for sensitive info
  • Watering hole — compromising sites target visits
  • Typosquatting — registering misspelled domain names
  • Impersonation — pretending to be someone else

Social Engineering Principles

  • Authority — people obey authority figures
  • Urgency — pressure to act quickly bypasses thinking
  • Intimidation — fear causes compliance
  • Consensus/Social proof — people follow the crowd
  • Scarcity — limited availability increases desire
  • Trust — exploiting established relationships
4
Application Attacks

Injection Attacks

  • SQL injection — inserting SQL commands into input fields
  • Command injection — executing OS commands through application
  • LDAP injection — manipulating LDAP queries
  • XML injection — inserting malicious XML
  • XSS (Cross-Site Scripting) — injecting client-side scripts
  • Stored XSS — script saved on server
  • Reflected XSS — script in URL, reflected back
  • DOM-based XSS — script manipulates page DOM

Memory Attacks

  • Buffer overflow — writing beyond allocated memory
  • Integer overflow — exceeding integer size limits
  • Memory leak — failing to release unused memory
  • Use after free — accessing memory after deallocation
  • Heap spray — filling heap with malicious code

Other Application Attacks

  • CSRF (Cross-Site Request Forgery) — forging requests from authenticated user
  • Directory traversal — accessing files outside web root (../)
  • Race condition — exploiting timing between operations
  • TOCTOU (Time of Check to Time of Use) — state change between check and use
  • API abuse — exploiting poorly secured APIs
  • Resource exhaustion — consuming all available resources
5
Denial of Service
  • DoS — Denial of Service, single source
  • DDoS — Distributed DoS, multiple sources (botnet)
  • SYN flood — exhausting connections with incomplete handshakes
  • Ping flood — overwhelming with ICMP echo requests
  • Ping of death — malformed ping packets
  • Smurf attack — ICMP amplification using broadcast
  • Fraggle attack — UDP amplification
  • Teardrop — overlapping IP fragments
  • Amplification attack — using protocols to magnify traffic
  • NTP amplification — abusing NTP monlist command
  • DNS amplification — abusing open DNS resolvers
  • Application layer DoS — targeting specific application
  • Slowloris — holding connections open slowly
6
Change Management

Change Management Concepts

  • Change management — controlled process for system modifications
  • Baseline — documented starting configuration
  • Configuration management — maintaining system state
  • Change Advisory Board (CAB) — oversees change process
  • Standard change — pre-approved, low-risk, routine
  • Normal change — requires CAB approval
  • Emergency change — urgent, expedited approval
  • Backout plan — reverting if change fails

Environments

  • Development — where code is written
  • Test/QA — where code is verified
  • Staging — mirrors production for final validation
  • Production — live systems with real data
  • Sandbox — isolated testing environment

Key Principles

  • Separation of duties — developers shouldn’t deploy to production
  • Incident management — restoring service after unplanned interruption
  • Problem management — identifying root causes
  • CMMI (Capability Maturity Model Integration) — process maturity levels
  • Level 1: Initial — ad hoc, chaotic
  • Level 2: Managed — project-level processes
  • Level 3: Defined — organization-wide processes
  • Level 4: Quantitatively Managed — metrics-driven
  • Level 5: Optimizing — continuous improvement

Lab Knowledge — Password Cracking

If you did the labs, you know this. If you didn’t, good luck.

Password Attack Types

  • Dictionary attack — trying words from a wordlist
  • Brute force — trying all possible combinations
  • Hybrid attack — dictionary + modifications (rules)
  • Rainbow table — precomputed hash lookup
  • Pass-the-hash — using hash without cracking
  • Credential stuffing — using leaked credentials on other sites
  • Password spraying — few passwords against many accounts

Windows Password Hashing

  • NTLM — NT LAN Manager, uses MD4
  • NTLM is unsalted — identical passwords = identical hashes
  • Vulnerable to rainbow tables because unsalted
  • Stored in SAM database or Active Directory

Linux Password Hashing

  • $1$ — MD5 crypt (weak)
  • $5$ — SHA-256 crypt
  • $6$ — SHA-512 crypt (current default)
  • Salt stored between second and third $ symbols
  • Key stretching — 5,000+ iterations by default
  • Stored in /etc/shadow
  • Much slower to crack than NTLM due to salting and iterations

1. You successfully crack 70% of a client’s Windows passwords in under an hour using a common wordlist. How do you present this finding to leadership?
2. A client’s users are falling for phishing emails despite training. What additional controls would you recommend?
3. You discover a client has no change management process—developers push directly to production. Explain the risk in business terms.
4. A penetration test reveals a buffer overflow vulnerability in a custom application. The development team says fixing it would take months. How do you advise the client?

Purple Team Thinking

5. You find unsalted password hashes in a database. Red team: what’s your attack plan? Blue team: what’s the remediation?
6. A client’s web application has no rate limiting on login attempts. Red team: how do you exploit this? Blue team: what controls are needed?
