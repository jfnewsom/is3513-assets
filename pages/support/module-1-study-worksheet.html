Malware & Attack Types
How the bad guys deliver payloads

Malware Categories

  • Virus — requires host file, user action to spread
  • Worm — self-replicating, no user action needed
  • Trojan — disguised as legitimate software
  • Ransomware — encryption, extortion, why modern versions are hard to reverse
  • Rootkit — hides at OS/kernel level
  • Spyware — monitors user activity
  • Keylogger — captures keystrokes
  • RAT (Remote Access Trojan) — persistent backdoor
  • Botnet — network of compromised systems
  • Logic bomb — triggered by condition
  • Fileless malware — lives in memory, no files on disk
Key question: What is the PRIMARY difference between a virus and a worm?

Social Engineering Attacks

  • Phishing — mass email, generic targets
  • Spear phishing — targeted at specific individual
  • Whaling — targeting executives
  • Business Email Compromise (BEC) — impersonating CEO, wire fraud
  • Vishing — voice phishing (phone calls)
  • Smishing — SMS phishing
  • Pretexting — fabricated scenario to extract info
  • Baiting — leaving infected USB drives
  • Tailgating / piggybacking — following through secure doors
  • Shoulder surfing — watching someone enter credentials
  • Dumpster diving — searching trash for sensitive info
  • Watering hole attack — compromising sites your target visits

Security Principles
The design rules behind every defense

These aren’t just definitions — understand WHY each principle matters.

Core Principles

  • Least privilege — minimum access necessary
  • Separation of duties — no single person controls entire process
  • Defense in depth / layered security — multiple overlapping controls
  • Need to know — access based on job requirements
  • Implicit deny / default deny — if not explicitly allowed, it’s denied
  • Zero Trust — never trust, always verify; validate every request
  • Open design — security shouldn’t depend on secrecy of mechanism
  • Security through obscurity — why it fails as primary defense
  • Fail-safe / fail-secure — system fails to secure state
  • Due diligence — research before acting
  • Due care — acting responsibly with what you know

Reconnaissance Concepts
Passive and active info-gathering techniques

Types of Reconnaissance

  • Passive reconnaissance — no direct interaction with target
  • Active reconnaissance — direct interaction, detectable
  • Footprinting — gathering info about target organization
  • Enumeration — extracting specific info (users, shares, services)
  • Banner grabbing — identifying service versions
  • OSINT (Open Source Intelligence) — publicly available info

Reconnaissance Techniques

  • WHOIS lookups — domain registration info
  • DNS reconnaissance — zone transfers, record enumeration
  • Google dorking — advanced search operators to find exposed info
  • Shodan — search engine for internet-connected devices
  • Metadata analysis — extracting info from document properties
  • Social media mining — OSINT from public profiles

Key Terms

  • CVE (Common Vulnerabilities and Exposures) — standardized vulnerability IDs
  • Exploit-DB — database of public exploits
  • Attack surface — all points where attacker can try to enter

Communication

Purple Team Thinking
