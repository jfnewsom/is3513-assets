Risk Management Fundamentals

Define each term. If you can’t explain it simply, you don’t understand it well enough.

Core Risk Concepts

  • Risk — possibility of suffering harm or loss
  • Threat — potential cause of unwanted incident
  • Vulnerability — weakness that can be exploited
  • Asset — anything of value to organization
  • Impact — actual damage when threat exploits vulnerability
  • Likelihood — probability threat will exploit vulnerability
  • Attack vector — path attacker uses to reach target
  • Threat vector — method threat uses to gain access

Risk Relationships

Risk = Threat × Vulnerability × Impact
or simplified: Risk = Likelihood × Impact

Risk Truths

  • Risk cannot be completely eliminated — it’s absolute
  • Not every risk needs to be mitigated
  • Risk management must balance cost vs. benefit
  • Qualitative and quantitative approaches both have value
2
Quantitative Risk Analysis

Key Formulas

  • SLE (Single Loss Expectancy) = AV × EF
  • ALE (Annualized Loss Expectancy) = SLE × ARO
  • AV = Asset Value
  • EF = Exposure Factor (percentage of asset lost)
  • ARO = Annualized Rate of Occurrence

Quantitative Metrics

  • TCO (Total Cost of Ownership) — all costs over asset lifetime
  • ROI (Return on Investment) — benefit vs. cost ratio
  • MTBF (Mean Time Between Failures) — average time before failure
  • MTTR (Mean Time to Repair) — average time to restore
  • RTO (Recovery Time Objective) — maximum acceptable downtime
  • RPO (Recovery Point Objective) — maximum acceptable data loss

Qualitative Risk Analysis

Qualitative Approach

  • Uses professional judgment and experience
  • Categorizes risks as High/Medium/Low
  • Risk matrix — plots likelihood vs. impact
  • 5×5 matrix — more granular than 3×3
  • Faster than quantitative, less precise
  • Subjective but valuable for prioritization

Risk Documentation

  • Risk register — documented list of identified risks
  • Risk owner — person accountable for managing specific risk
  • Risk appetite — level of risk organization willing to accept
  • Risk tolerance — acceptable deviation from risk appetite
  • Residual risk — risk remaining after controls implemented
  • Inherent risk — risk before any controls
4
Risk Treatment Strategies

The Four Treatment Options

  • Mitigate/Reduce — implement controls to reduce likelihood or impact
  • Transfer — shift risk to third party (insurance, outsourcing)
  • Accept — acknowledge risk and continue without action
  • Avoid — eliminate risk by discontinuing activity

Treatment Decisions

  • Cost of control should not exceed potential loss
  • Accept when control costs exceed risk impact
  • Transfer when risk is outside core competency
  • Avoid when risk is unacceptable and no reasonable control exists
  • Cyber insurance — example of risk transfer
5
Vulnerability Management

Vulnerability Sources

  • CVE (Common Vulnerabilities and Exposures) — standardized IDs
  • NVD (National Vulnerability Database) — NIST-maintained database
  • Exploit-DB — public exploit database
  • Vendor advisories — Microsoft, Cisco, etc.
  • Bug bounty programs — crowdsourced vulnerability discovery

CVSS (Common Vulnerability Scoring System)

  • Version 3.1 is current standard
  • Scores range 0.0 to 10.0
  • None: 0.0 • Low: 0.1–3.9 • Medium: 4.0–6.9 • High: 7.0–8.9 • Critical: 9.0–10.0

Vulnerability Scanning

  • Credentialed scan — authenticates to target, more thorough
  • Non-credentialed scan — external perspective only
  • Active scanning — sends probes to target
  • Passive scanning — observes traffic without probing
  • False positive — reported vulnerability that doesn’t exist
  • False negative — missed actual vulnerability
6
Infrastructure Security

Network Security Devices

  • Firewall — implicit deny, rule-based filtering
  • Next-gen firewall (NGFW) — application awareness, IPS, deep inspection
  • WAF (Web Application Firewall) — protects web apps
  • UTM (Unified Threat Management) — multiple security functions
  • NAC (Network Access Control) — verifies device compliance

Data Protection

  • DLP (Data Loss Prevention) — prevents unauthorized data transfer
  • Encryption at rest — stored data encrypted
  • Encryption in transit — data encrypted during transmission
  • Data classification — categorizing data by sensitivity
  • Data masking — obscuring sensitive data
  • Tokenization — replacing sensitive data with tokens

Virtualization Security

  • Hypervisor — Type 1 (bare metal) vs. Type 2 (hosted)
  • VM escape — breaking out of VM to host
  • VM sprawl — uncontrolled VM proliferation
  • Snapshot — point-in-time VM state
  • Resource isolation — preventing VMs from affecting each other

Network Attacks (Review)

  • ARP poisoning — falsifying ARP responses
  • MAC flooding — overwhelming switch CAM table
  • VLAN hopping — gaining access to other VLANs
  • DNS poisoning — corrupting DNS cache
