Cryptographic Foundations
Symmetric, asymmetric, and the data states they protect

Define each term. If you can’t explain it simply, you don’t understand it well enough.

Core Cryptographic Goals

  • Confidentiality — keeping data secret
  • Integrity — ensuring data hasn’t been altered
  • Authentication — verifying identity
  • Non-repudiation — proof of origin that cannot be denied

Data States

  • Data at rest — stored on disk, database, backup
  • Data in transit — moving across network
  • Data in use — currently being processed by CPU

Encryption Types

  • Symmetric encryption — same key encrypts and decrypts
  • Asymmetric encryption — public key encrypts, private key decrypts (or vice versa for signing)
  • Stream cipher — encrypts one bit/byte at a time
  • Block cipher — encrypts fixed-size blocks

Symmetric Algorithms

  • AES (Advanced Encryption Standard) — current NIST standard, 128/192/256-bit keys
  • DES (Data Encryption Standard) — obsolete, 56-bit key
  • 3DES (Triple DES) — legacy, applies DES three times
  • Blowfish / Twofish — alternative symmetric ciphers
  • RC4 — stream cipher, considered weak

Asymmetric Algorithms

  • RSA — most common, used for encryption and digital signatures
  • Diffie-Hellman — key exchange only, not encryption
  • ECC (Elliptic Curve Cryptography) — smaller keys, equivalent security
  • DSA (Digital Signature Algorithm) — signatures only
  • Ed25519 — modern elliptic curve for SSH keys

Cipher Modes

  • CBC (Cipher Block Chaining) — each block XORed with previous
  • GCM (Galois/Counter Mode) — provides authentication + encryption
  • ECB (Electronic Codebook) — insecure, patterns visible
2
Hashing
Integrity, password storage, and known attacks

Hash Function Properties

  • One-way function — cannot reverse hash to get input
  • Deterministic — same input always produces same output
  • Collision resistance — two different inputs should not produce same hash
  • Avalanche effect — small input change causes drastically different output
  • Fixed output length — regardless of input size

Hash Algorithms

  • MD5 — 128-bit (32 hex chars), broken, do NOT use for security
  • SHA-1 — 160-bit (40 hex chars), deprecated, vulnerable to collisions
  • SHA-256 — 256-bit (64 hex chars), current standard
  • SHA-512 — 512-bit (128 hex chars), stronger variant
  • SHA-3 — newest SHA family, different internal structure

Password Hashing

  • Salting — random data added before hashing to prevent rainbow tables
  • Key stretching — running hash algorithm many times to slow cracking
  • bcrypt — adaptive password hashing with built-in salt
  • PBKDF2 — key derivation function with configurable iterations
  • Argon2 — modern password hashing, memory-hard

Attacks on Hashing

  • Rainbow table attack — precomputed hash lookup table
  • Birthday attack — exploits collision probability mathematics
  • Dictionary attack — trying common passwords
  • Brute force — trying all possible combinations
3
Digital Signatures & PKI
How trust gets established across the internet

Digital Signatures

  • Created with sender’s private key
  • Verified with sender’s public key
  • Provides authentication, integrity, and non-repudiation
  • Process: hash message → encrypt hash with private key

Public Key Infrastructure (PKI)

  • Certificate Authority (CA) — trusted third party that issues certificates
  • Digital certificate — binds public key to identity
  • Certificate chain — hierarchy from root CA to end certificate
  • Certificate revocation — invalidating compromised certificates
  • CRL (Certificate Revocation List) — list of revoked certificates
  • OCSP (Online Certificate Status Protocol) — real-time revocation checking

Certificate Types

  • Self-signed certificate — signed by own private key, not trusted by browsers
  • CA-signed certificate — signed by trusted CA, browser-trusted
  • Wildcard certificate — covers all subdomains (*.example.com)
  • SAN certificate — Subject Alternative Name, multiple domains
4
Authentication
Factors, MFA, and the protocols behind login

Authentication Factors

  • Something you know — password, PIN, security questions
  • Something you have — smart card, token, phone (SMS/app)
  • Something you are — fingerprint, facial recognition, iris scan
  • Somewhere you are — geolocation, IP-based
  • Something you do — typing patterns, gait analysis

Multi-Factor Authentication (MFA)

  • Combines two or more different factor types
  • Password + SMS code = 2FA (but SMS is weak)
  • Password + authenticator app = stronger 2FA
  • TOTP (Time-based One-Time Password) — changes every 30 seconds
  • HOTP (HMAC-based One-Time Password) — counter-based

Authentication Protocols

  • PAP (Password Authentication Protocol) — cleartext, insecure
  • CHAP (Challenge-Handshake Authentication Protocol) — challenge-response, more secure
  • EAP (Extensible Authentication Protocol) — framework for various methods
  • Kerberos — ticket-based, uses KDC (Key Distribution Center)

AAA Framework

  • Authentication — proving who you are
  • Authorization — what you’re allowed to do
  • Accounting — logging what you did

AAA Protocols

  • RADIUS — UDP, encrypts only password, common for network access
  • TACACS+ — TCP, encrypts entire payload, Cisco-developed
  • 802.1X — port-based network access control using EAP

Access Control Models

  • DAC (Discretionary Access Control) — owner controls access
  • MAC (Mandatory Access Control) — system-enforced labels
  • RBAC (Role-Based Access Control) — permissions based on job role
  • ABAC (Attribute-Based Access Control) — policies based on attributes

Single Sign-On (SSO)

  • Authenticate once, access multiple applications
  • SAML — XML-based, enterprise SSO
  • OAuth — authorization framework (not authentication)
  • OpenID Connect — authentication layer on top of OAuth
  • Federation — trust between different organizations
5
Secure Protocols
SSH, TLS, and the rest of the secure-channel toolkit

SSH (Secure Shell)

  • Encrypted remote access — replaces Telnet
  • Port 22 by default
  • Supports password and key-based authentication
  • Key-based auth stronger than passwords

TLS/SSL

  • TLS (Transport Layer Security) — current standard, replaced SSL
  • SSL (Secure Sockets Layer) — deprecated, do not use
  • HTTPS — HTTP over TLS (port 443)
  • Cipher suite — combination of key exchange, encryption, and hash algorithms
  • Perfect Forward Secrecy — compromised key doesn’t expose past sessions

Other Secure Protocols

  • SFTP — SSH File Transfer Protocol
  • FTPS — FTP over TLS
  • LDAPS — LDAP over TLS
  • DNSSEC — DNS Security Extensions
  • S/MIME — secure email

Cryptographic Attacks
How attackers break weak crypto
  • Man-in-the-Middle (MITM) — attacker intercepts and possibly modifies communication
  • Replay attack — capturing and retransmitting valid data
  • Downgrade attack — forcing use of weaker encryption
  • Side-channel attack — exploiting implementation (timing, power analysis)
  • Known plaintext attack — attacker has plaintext and ciphertext pairs
  • Chosen plaintext attack — attacker can encrypt arbitrary plaintexts
  • Pass-the-hash — using stolen hash without cracking it
