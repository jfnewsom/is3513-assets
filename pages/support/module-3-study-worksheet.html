Network Fundamentals

Define each term. If you can’t explain it simply, you don’t understand it well enough.

The OSI Model

  • Layer 7 — Application (HTTP, FTP, SMTP)
  • Layer 6 — Presentation (encryption, compression)
  • Layer 5 — Session (session management)
  • Layer 4 — Transport (TCP, UDP)
  • Layer 3 — Network (IP, routing)
  • Layer 2 — Data Link (MAC addresses, switches)
  • Layer 1 — Physical (cables, hubs)

TCP vs. UDP

  • TCP — connection-oriented, reliable, three-way handshake (SYN, SYN/ACK, ACK)
  • UDP — connectionless, unreliable, no handshake, faster
  • TCP ports — HTTP (80), HTTPS (443), SSH (22), FTP (21), Telnet (23), SMTP (25)
  • UDP ports — DNS (53), DHCP (67/68), TFTP (69), SNMP (161)

Network Protocols

  • ICMP — Internet Control Message Protocol, used by ping and traceroute
  • ARP — Address Resolution Protocol, maps IP to MAC addresses
  • DHCP — Dynamic Host Configuration Protocol, assigns IP addresses
  • DNS — Domain Name System, resolves names to IP addresses
  • SLAAC — Stateless Address Autoconfiguration, IPv6 auto-config

IP Addressing

  • IPv4 — 32 bits, dotted decimal (192.168.1.1)
  • IPv6 — 128 bits, hexadecimal with colons
  • Private IP ranges — 10.x.x.x, 172.16–31.x.x, 192.168.x.x
  • Loopback — 127.0.0.1 (IPv4), ::1 (IPv6)
  • CIDR notation — /24 = 255.255.255.0

Network Classes (Legacy)

  • Class A — /8, 16 million hosts
  • Class B — /16, 65,534 hosts
  • Class C — /24, 254 hosts
2
Network Devices

Layer 1 Devices

  • Hub — broadcasts to all ports, shared collision domain
  • Repeater — regenerates signal to extend distance

Layer 2 Devices

  • Switch — forwards based on MAC address, separate collision domains per port
  • Bridge — connects network segments at Layer 2
  • WAP (Wireless Access Point) — connects wireless clients to wired network

Layer 3 Devices

  • Router — forwards based on IP address, separates broadcast domains
  • Layer 3 switch — switch with routing capabilities

Security Devices

  • Firewall — filters traffic based on rules
  • IDS (Intrusion Detection System) — monitors and alerts
  • IPS (Intrusion Prevention System) — monitors and blocks
  • Proxy — intermediary that filters/caches traffic
  • Load balancer — distributes traffic across servers
  • VPN concentrator — handles VPN connections
3
Network Architecture

Network Types

  • LAN — Local Area Network, single location
  • WAN — Wide Area Network, geographically dispersed
  • WLAN — Wireless LAN
  • MAN — Metropolitan Area Network
  • PAN — Personal Area Network (Bluetooth)

Network Zones

  • Intranet — internal network, trusted zone
  • Extranet — extended access to partners
  • DMZ — Demilitarized Zone, buffer between internal and external
  • Internet — public, untrusted network

Topologies

  • Star — all devices connect to central point (most common)
  • Bus — single cable, all devices attached
  • Ring — devices connected in a circle
  • Mesh — every device connected to every other device
  • Hybrid — combination of topologies

Segmentation

  • VLAN — Virtual LAN, logical segmentation
  • Subnetting — dividing networks into smaller segments
  • NAT — Network Address Translation, hides internal IPs
  • PAT — Port Address Translation, multiple internal IPs share one external
  • Air gap — complete physical isolation
4
Intrusion Detection & Prevention

IDS Types

  • NIDS — Network-based, monitors network traffic
  • HIDS — Host-based, monitors single system (files, processes, logs)
  • Wireless IDS — monitors wireless networks

Detection Methods

  • Signature-based — compares against known attack patterns (signature database)
  • Anomaly-based — compares against baseline of normal behavior
  • Heuristic — uses rules and algorithms to identify suspicious activity

IDS Outcomes

  • True positive — correctly identified attack
  • True negative — correctly identified normal traffic
  • False positive — alert on legitimate traffic (not actually malicious)
  • False negative — missed actual attack (malicious goes undetected)

IDS/IPS Tools

  • Snort — open-source IDS since 1998, de facto standard
  • Suricata — high-performance alternative to Snort
  • OSSEC — host-based IDS
  • Zeek (formerly Bro) — network analysis framework

Key Concepts

  • Promiscuous mode — NIC captures all traffic, not just addressed to it
  • Inline vs. passive — IPS inline (can block), IDS passive (monitor only)
  • Correlation — connecting related events from multiple sources
  • SIEM — Security Information and Event Management
  • Encrypted traffic limitation — NIDS cannot inspect encrypted payloads
5
Cloud & Virtualization

Cloud Service Models

  • IaaS — Infrastructure as a Service (VMs, storage, networking)
  • PaaS — Platform as a Service (development environment)
  • SaaS — Software as a Service (applications via browser)
  • XaaS — Anything as a Service

Cloud Deployment Models

  • Public cloud — shared infrastructure, third-party operated
  • Private cloud — dedicated to single organization
  • Hybrid cloud — combination of public and private
  • Community cloud — shared by organizations with common concerns

Virtualization

  • Hypervisor — software layer between VMs and hardware
  • Type 1 (bare metal) — runs directly on hardware (VMware ESXi, Hyper-V)
  • Type 2 (hosted) — runs on host OS (VirtualBox, VMware Workstation)
  • VM sprawl — uncontrolled proliferation of virtual machines
  • VM escape — attack breaks out of VM to host
  • Snapshot — saved state of VM at point in time

Cloud Security

  • Shared responsibility model — cloud provider vs. customer responsibilities
  • CASB — Cloud Access Security Broker
  • Data sovereignty — data subject to laws of country where stored

Network Attacks
  • ARP poisoning/spoofing — falsifying ARP responses to redirect traffic
  • MAC flooding — overwhelming switch CAM table to force hub behavior
  • VLAN hopping — gaining access to other VLANs
  • DNS poisoning — corrupting DNS cache with false records
  • DHCP starvation — exhausting DHCP address pool
  • Rogue DHCP — unauthorized DHCP server assigning malicious settings
  • Man-in-the-Middle — intercepting communication between two parties
  • Replay attack — capturing and retransmitting valid traffic
  • Amplification attack — using protocols to magnify DoS traffic

Professional Application

Ethics, communication, and judgment matter as much as technical skills.

1. Your IDS is generating hundreds of alerts per day, most of which are false positives. How do you approach tuning it?
2. A client asks whether they should use signature-based or anomaly-based detection. What factors influence your recommendation?
3. You capture network traffic and find unencrypted credentials. How do you report this finding?
4. A client’s network has no segmentation—everything is on one flat network. Explain the risk in business terms.

Purple Team Thinking

5. You discover a switch with no port security. Red team: what attack does this enable? Blue team: what’s the mitigation?
6. A client’s DMZ allows direct database connections from the internet. Red team: why is this attractive? Blue team: what architecture change is needed?
