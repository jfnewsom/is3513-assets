Why 10 Citations?

Each module has 18-26 natural citation opportunities. If you're actually documenting your methodology, 10 is trivially easy. The requirement ensures you're engaging with the material, not just running commands blindly.

Translation: If you're struggling to find 10 citations, you're not doing the research that makes your work credible.
What to Cite
Strong Sources
  • Official tool documentation
  • Man pages you referenced
  • Vendor security guides
  • NIST, CIS, OWASP standards
  • RFC documents
  • Your course textbook
  • Peer-reviewed papers
Acceptable Sources
  • Reputable tech blogs
  • Stack Overflow answers
  • GitHub documentation
  • Tutorial sites (with caution)
  • Wikipedia (for general context)
Avoid: Random forum posts, AI-generated content, sources you didn't actually read, anything behind a paywall you didn't access.
How to Cite (Using Word)

Use Word's built-in citation tools — don't type references manually.

  1. Place cursor where you want the citation
  2. Go to References → Insert Citation → Add New Source
  3. Fill in the source details (author, title, URL, date accessed)
  4. At the end of your document: References → Bibliography → Insert Bibliography
  5. Before submitting: Select all (Ctrl+A) and update fields (F9)
Style: Use APA format. Consistency matters.

Pro Tip: Cite as You Work
Keep a running list of sources as you work through each lab. When you look something up, add it immediately. Trying to reconstruct your sources after the fact is painful and usually incomplete.
