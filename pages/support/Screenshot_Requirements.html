Evidence Standards for Lab Submissions

Screenshots are evidence. In professional security work, undocumented claims don’t exist. Every screenshot must prove you did the work on your system.

Deduction: −5% per missing or unverifiable screenshot. No VM name visible? Wrong filename? Missing required capture? Each one costs you.
✓ Verifiable Screenshot
VM name in terminal prompt — kali-lastname-abc123 visible
VM name in window title bar — hostname visible at top
Hostname visible in command output — e.g., uname -n output
✗ Unverifiable Screenshot
Generic "kali@kali" prompt — default name, not yours
Cropped to hide hostname — partial screenshots rejected
Web browser only, no VM context — could be anyone’s machine
Critical: VM Name Visibility

Your VM must be named kali-lastname-studentID and this name must be visible in every screenshot.

Without your VM name, there’s no way to verify you ran the commands. Screenshots without identification could be copied from anywhere — and occasionally are.

Filename Convention

All screenshots follow this naming pattern: ##-description.png

  • 01-vm-settings.png — First screenshot, VM configuration
  • 07-nmap-scan-results.png — Seventh screenshot, nmap output
  • 15-wireshark-tcp-stream.png — Fifteenth screenshot, Wireshark analysis

Two-digit numbers keep files sorted correctly (01, 02… not 1, 2, 10, 11). This matters when you’re compiling 20+ screenshots into a report.

Folder Organization

Keep screenshots organized by lab in your Kali VM:

~/Screenshots/ ├── Lab1-Screenshots/ │ ├── 01-vm-settings.png │ ├── 02-kali-desktop.png │ └── ... ├── Lab2-Screenshots/ ├── Lab3-Screenshots/ └── ...
Pro Tip: Screenshot as you go. Don’t try to recreate screenshots after the fact. Take them in real-time as you work. Missing a screenshot means redoing the work — or taking the deduction.
