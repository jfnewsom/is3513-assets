Screenshots are evidence. In professional security work, undocumented claims don't exist. Every screenshot must prove YOU did the work on YOUR system.

-5 pts
Per Missing or Unverifiable Screenshot
No VM name visible? Wrong filename? Missing required capture? Each one costs you.
Critical: VM Name Visibility

Your VM must be named kali-lastname-studentID and this name must be visible in every screenshot.

Verifiable Screenshot
  • VM name in terminal prompt
  • VM name in window title bar
  • Hostname visible in output
Unverifiable Screenshot
  • Generic "kali@kali" prompt
  • Cropped to hide hostname
  • Web browser only (no VM context)
Why? Without your VM name, I can't verify YOU ran the commands. Screenshots without identification could be copied from anywhere.
Filename Convention

All screenshots follow this naming pattern:

##-description.png
Example Explanation
01-vm-settings.png First screenshot, VM configuration
07-nmap-scan-results.png Seventh screenshot, nmap output
15-wireshark-tcp-stream.png Fifteenth screenshot, Wireshark analysis
Two-digit numbers keep files sorted correctly (01, 02... not 1, 2, 10, 11).
What to Capture
Always Include
  • Command you ran
  • Output/results
  • VM name (terminal prompt or title)
  • Relevant context (IP addresses, etc.)
When Instructed
  • Error messages (document failures too)
  • Configuration screens
  • Before/after comparisons
  • Specific UI elements
Folder Organization

Keep screenshots organized by lab in your Kali VM:

~/Screenshots/
├── Lab1-Screenshots/

│   ├── 02-kali-desktop.png

Pro Tip: Screenshot as You Go
Don't try to recreate screenshots later. Take them in real-time as you work through each lab. Missing a screenshot means redoing the work — or taking the deduction.
